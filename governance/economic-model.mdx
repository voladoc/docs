## Circular Economy Design

<Card>
  VOLA Protocol implements a circular economic design where value flows through the system in a self-reinforcing loop.
</Card>

<Steps>
  <Step title="Value Creation">
    Users interact with protocol products (Wave Protocol, hbHYPE), generating fees and creating utility.
  </Step>
  <Step title="Value Capture">
    Fees are collected and distributed according to the revenue distribution model.
  </Step>
  <Step title="Value Reinvestment">
    A portion of revenue is reinvested in protocol development, security, and growth.
  </Step>
  <Step title="Stakeholder Rewards">
    Value flows to veVOLA holders and system participants, aligning incentives.
  </Step>
  <Step title="Enhanced Participation">
    Rewards incentivize increased protocol usage and participation.
  </Step>
  <Step title="Expanded Value Creation">
    Growth leads to more usage, starting the cycle again at a larger scale.
  </Step>
</Steps>

<Frame>
  ![Circular Economy Flow](/images/circular-economy.png)
</Frame>

<Warning>
  This circular design requires careful balancing to avoid inflationary pressures while ensuring sufficient rewards to drive participation and growth.
</Warning>

## Economic Parameters

<Card>
  The protocol's economic model is governed by several key parameters that can be adjusted through governance to adapt to changing conditions.
</Card>

### Key Economic Parameters

| Parameter | Initial Value | Description | Governance Control |
| :-------- | :------------ | :---------- | :---------------- |
| Base Minting Fee | 0.5% | Standard fee for creating synthetic assets | Full |
| Base Redemption Fee | 0.3% | Standard fee for redeeming synthetic assets | Full |
| Fee Distribution Allocation | 30/50/15/5 | Breakdown of revenue distribution | Limited¹ |
| veVOLA Boost Factor | 2.5x max | Maximum reward boost for veVOLA holders | Limited¹ |
| Emission Rate | Varies by phase | Rate of token distribution from incentives | Partial² |
| Rebalancing Pool Reward Rate | 50% of fees | Percentage of fees allocated to stability providers | Full |
| Flash Loan Fee | 0.05% | Fee for instantaneous borrowing | Full |
| Insurance Fund Utilization | Emergency only | Conditions for using insurance reserves | Multi-sig³ |

<Note>
  ¹Limited: Changes require super-majority and extended timelock
  ²Partial: Within governance-approved bounds
  ³Multi-sig: Requires approval from security council and governance
</Note>
